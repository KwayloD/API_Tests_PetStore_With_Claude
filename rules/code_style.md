# Соглашения по именованию и стилю

## Файлы тестов

| Что | Шаблон | Пример |
|-----|--------|--------|
| Файл | `test_{ресурс}_{тип}.py` | `test_pets_positive.py` |
| Класс | `Test{Ресурс}{Тип}` | `TestPetsPositive` |
| Метод | `test_{действие}_{условие}` | `test_get_pet_not_found` |

## Фикстуры

| Что | Шаблон | Пример |
|-----|--------|--------|
| Payload | `{ресурс}_payload` | `pet_payload` |
| С созданием ресурса | `created_{ресурс}` | `created_pet` |
| Mock | `mock_{описание}` | `mock_inventory` |

## Типы тестов

- **Позитивные** (`_positive`) — успешные сценарии с валидными данными
- **Негативные** (`_negative`) — обработка ошибок (404, невалидные входные данные)
- **Мок** (`_mock`) — изолированные тесты без реальных HTTP-запросов; используются для сценариев, сложно воспроизводимых на живом API (500, несовпадение ID и т.п.)

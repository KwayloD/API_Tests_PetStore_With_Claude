# Особенности публичного Petstore API

Реальное поведение API расходится со Swagger-документацией:

| Запрос | Ожидалось по Swagger | Реальный ответ |
|--------|----------------------|----------------|
| `GET /pet/{строка}` | 400 | **404** (NumberFormatException на сервере) |
| `GET /store/order/{строка}` | 400 | **404** (то же самое) |
| `PUT /user/{несуществующий}` | 404 | **200** — делает upsert, создаёт пользователя |
| `GET /user/login` с пустыми данными | 400 | **200** — валидация credentials отсутствует |

Эти quirks уже учтены в тестах и задокументированы в комментариях к соответствующим методам.
